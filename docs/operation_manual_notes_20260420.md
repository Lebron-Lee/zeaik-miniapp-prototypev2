# 智爱客小程序操作手册撰写笔记

## 1. 角色范围确认

当前原型需要在操作手册中覆盖的角色，至少包括外部流量用户、顾问演示者、普通员工、门店管理者、区域/集团管理者，以及通过邀请链接进入的新成员。之所以需要这样划分，是因为同一个小程序通过不同入口与不同状态承接不同身份，且培训、组织与管理操作并不是完全割裂的独立系统。[1] [2] [3]

| 角色 | 进入方式 | 主要目标 | 核心页面或区域 |
|---|---|---|---|
| 外部流量用户 | 广告、会销、扫码、分享链接 | 了解产品、提问、决定是否试用 | 首页、视频介绍页、产品介绍页、店铺信息页 |
| 顾问演示者 | 会议演示、销售讲解 | 现场演示产品价值与培训闭环 | 首页、产品介绍页、当前工作页、组织选择页 |
| 普通员工 | 登录进入、培训链接进入 | 查看工作、参加培训、查看工资、参与巡检 | 当前工作页、抽屉培训中心、工资日结、AI 巡检 |
| 门店管理者 | 登录后从首页或培训入口进入 | 发起培训、选择对象、查看进度与回看历史 | 首页、组织选择页、当前工作页、抽屉培训中心、培训详情 |
| 区域/集团管理者 | 管理端身份进入 | 查看组织层级、跟踪培训进展、做跨层级观察 | 组织选择页、抽屉培训中心、集团 AI 页面 |
| 新成员 / 被邀请人 | 培训邀请链接、二维码 | 无感注册、绑定组织关系、接收培训 | 邀请注册页、当前工作页 |

## 2. 全局导航与状态模型

小程序采用单页壳层切换方式，顶层通过 `MiniAppShell.tsx` 统一管理 `home`、`current-task`、`daily-salary`、`inspection`、`product`、`video`、`org-tree`、`org-register` 等视图。虽然代码中仍保留 `training`、`training-answer`、`training-report`、`training-manager` 等旧演示页，但当前真实业务主链已收敛到“首页发起 + 当前工作页完成培训 + 抽屉沉淀历史”的结构，因此操作手册应以真实主链为准，旧页只可作为补充说明，不应写成主流程。[2] [3]

| 状态或入口 | 作用 | 手册写法 |
|---|---|---|
| Stage 1 游客 | 免登录体验首页对话、视频、产品介绍 | 归入外部流量用户操作说明 |
| Stage 2 已登录 | 解锁工资、当前工作、培训、巡检等功能 | 归入员工/管理者操作说明 |
| Stage 3 深度转化 | 填写店铺信息、申请试用 | 归入顾问/意向客户操作说明 |
| 首页培训入口 | 快速发起培训业务 | 归入管理者与顾问演示者说明 |
| 当前工作页 | 培训主执行场 | 归入员工主流程说明 |
| 抽屉培训中心 | 历史任务、进展追踪、再次进入任务 | 归入员工与管理者共用说明 |

## 3. 员工真实培训主链

员工的真实培训链路已经迁移到当前工作页。该页顶部保持工作台身份栏，中间直接展示培训任务卡，用户点击“立即培训”后会话气泡就在同页继续追加，底部输入框同步开启。答错后页面不会跳出，而是在同一消息流中提示继续思考、查看提示或跳过此题。全部题目完成后，系统继续收集一句主观反馈，再在当前页内联展示培训完成结果卡，并把结果同步到任务档案。[4]

| 步骤 | 页面位置 | 用户动作 | 系统反馈 |
|---|---|---|---|
| 看到任务 | 当前工作页训练面板顶部 | 查看培训标题、时间、按钮状态 | 显示“立即培训/继续培训/已完成” |
| 开始培训 | 当前工作页同页 | 点击“立即培训” | 立即在消息流插入第 1 题 |
| 回答问题 | 当前工作页底部输入区 | 输入文字并发送 | 系统判断是否正确，进入追问或下一题 |
| 答错处理 | 当前工作页同页 | 继续作答、查看提示、跳过此题 | 显示追问卡、提示块或跳过提示 |
| 完成全部题目 | 当前工作页同页 | 提交最后反馈 | 显示完成卡、综合得分、星级与结果已同步 |
| 历史回看 | 抽屉培训中心 / 会话记录 | 打开历史任务或相关会话 | 返回对应任务详情或当前任务上下文 |

## 4. 抽屉培训中心承接逻辑

培训结果与管理动作不再以独立培训报告页或独立培训管理页作为主链承接，而是更多沉淀在抽屉中的培训中心与会话记录。培训中心按“我发起的”和“我参加的”分组，支持查看全部、查看新进度、进入指定任务；会话记录则承接与培训相关的历史会话。对管理者而言，这里是跟踪进度、查看变化、再次进入详情的地方；对普通员工而言，这里是找回历史培训与继续任务的地方。[5]

## 5. 管理者发起培训链路

管理者主要从首页培训入口发起培训，再进入组织选择页完成对象确认。组织选择页是真实可开发的组织对象选择器，第一层先选集团、管理层、店长等层级，第二层再选成员或业务组，支持搜索、部门全选、业务组全选与成员多选。只有至少选中一名成员，底部确认栏才有意义；确认后，选中结果回流到前一发起上下文。页面中同时保留“邀请成员”入口，用于把未注册成员通过邀请链接接入组织体系。[6]

| 管理动作 | 页面 | 关键规则 |
|---|---|---|
| 进入培训发起 | 首页培训入口 | 强调快速启动，不在首页完成全部配置 |
| 选择组织层级 | 组织选择页一级列表 | 先选集团/管理层/店长等对象层级 |
| 选择成员或业务组 | 组织选择页二级列表 | 支持按业务组批量勾选，也支持单人勾选 |
| 确认培训对象 | 底部确认栏 | 至少选中 1 名成员后再回流 |
| 邀请未注册成员 | 邀请入口 | 跳转到邀请注册承接流程 |
| 查看进展 | 抽屉培训中心 / 培训详情 | 通过“我发起的”任务查看最新进度 |

## 6. 新成员邀请与加入组织流程

新成员通过邀请链接进入后，需要在低阻力注册流程中完成最小信息补充。当前注册页采用 3 步式引导：第一步填写姓名与手机号；第二步选择职位、所属门店与直属上级，并勾选协议；第三步展示加入成功结果。若链接中带有邀请人信息，则页面会自动预填上级信息，减少填写成本。该流程适用于被邀请的普通员工，也适用于组织向上扩展时新增管理者。[7]

## 7. 其他模块的角色归属

除培训外，操作手册还应覆盖工资日结、AI 巡检、产品介绍、视频介绍、店铺信息采集、AI 菜单、店面 AI、集团 AI 等模块。这些模块多数不属于复杂流程页，但它们分别支撑员工日常查看、顾问演示、外部转化与集团能力理解，因此不能在手册中缺失。[1] [2] [3]
