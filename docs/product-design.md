## 1. 产品概述

智爱客（Zeaik）是面向餐饮行业的 AI 赋能小程序，核心价值在于通过 AI 对话降低员工培训门槛、提升门店运营效率。产品以微信小程序为主要载体，采用「对话即服务」的设计理念，将工资查询、任务管理、AI 巡检、碎片化培训等功能统一融入 AI 对话流中。

**核心差异点：**

- 培训以 AI 对话流形式进行，员工无需离开对话界面即可完成答题
- 组织架构通过「通用邀请链接」自然生长，接收者自行选择与邀请人的层级关系
- 管理者无需主动维护组织树，员工注册后自动归位

## 2. 用户角色体系

系统中存在三类核心角色，其权限和功能访问范围存在明显差异。

| 角色 | 典型用户 | 核心功能 | 特殊权限 |
|------|----------|----------|----------|
| **普通员工** | 服务员、厨师、收银员 | 查工资、看任务、接受培训、AI 对话 | 无 |
| **门店管理者** | 店长、副店长 | 全部员工功能 + 发起培训 + 查看团队报告 + 管理组织架构 | 可发起培训任务、查看问题清单、邀请成员 |
| **区域/集团管理者** | 区域经理、总部管理层 | 全部管理功能 + 跨门店数据汇总 + 集团 AI 模型配置 | 可查看下级门店培训数据、配置集团 AI 模型 |

**角色识别机制：** 当前版本通过 `isManager` prop 控制管理功能入口的显隐，真实场景中应通过登录后的用户 profile 中的 `role` 字段判断。

**角色状态说明：**

- `hasSuperior: false`：当前用户尚未绑定上级，为「孤立节点」状态，发起培训时系统提示绑定上级
- `hasSuperior: true`：已绑定上级，培训数据可向上同步，上级可在管理端查看下级团队数据
- `isFirstLaunch: true`：首次发起培训，系统在步骤 1 展示「上级数据同步」引导卡

## 3. 全局状态机

### 3.1 用户阶段（UserStage）

小程序将用户访问分为三个阶段，每个阶段决定可访问的功能范围。

```
Stage 1（游客）──登录──> Stage 2（已登录）──深度交流──> Stage 3（意向用户）
```

| 阶段 | 触发条件 | 可用功能 | 锁定功能 |
|------|----------|----------|----------|
| **Stage 1** | 从广告/分享链接进入，未登录 | AI 对话（受限）、产品介绍、视频介绍 | 工资日结、AI 巡检、培训、任务管理 |
| **Stage 2** | 完成微信一键登录 | 全部功能 | 无 |
| **Stage 3** | 在首页 AI 对话中完成深度交流后触发 | 全部功能 + 店铺信息采集 | 无 |

### 3.2 页面视图路由（AppView）

全局路由通过 `appView` 状态字符串控制，所有页面以「单页切换」方式渲染（非 URL 路由），适配微信小程序的页面栈模型。

```
home
├── video
├── store-info
├── product
├── current-task
│   └── quota-detail
├── inspection
├── daily-salary
├── ai-menu
├── store-ai-model
├── group-ai-model
└── training
    ├── training-answer（已废弃，答题融入对话流）
    ├── training-report
    ├── training-manager
    │   └── org-tree
    │       └── org-register
```

> **注意：** `training-answer` 视图在代码中保留但已不再通过正常路径触发，答题流程已完全内嵌于 `training` 页面的对话流中。

### 3.3 登录弹窗状态

登录弹窗（`LoginModal`）以覆盖层形式出现，不切换 `appView`。触发条件：

- 用户点击需要登录的功能入口（工资日结、AI 巡检、培训等）时，若 `isLoggedIn === false`
- 触发时携带 `trigger` 参数（如「工资日结」），弹窗标题显示「使用「工资日结」需要先登录」

## 4. 页面模块详解

### 4.1 登录与身份认证

**页面：** `LoginPage`、`LoginModal`（弹窗）

**两种登录入口：**

1. **独立登录页**（`LoginPage`）：用于首次进入小程序时的全屏登录引导，包含协议勾选、微信一键登录按钮。
2. **登录弹窗**（`LoginModal`）：从首页功能入口触发，以底部上滑弹窗形式出现，不打断用户当前浏览上下文。

**状态流转：**

```
未登录 ──点击微信一键登录──> 登录成功（isLoggedIn=true, stage=2）
                          ──点击「暂不登录」──> 关闭弹窗，保持 Stage 1
```

**关键字段：**

- `userPhone`：登录后存储手机号（当前 Mock 为 `138****8888`），用于店铺信息页展示
- `agreed`：是否勾选用户协议，未勾选时登录按钮不可点击

---

### 4.2 首页（AI 对话中枢）

**页面：** `HomePage`

首页是产品的核心交互界面，承载 AI 对话、功能入口导航、问题清单三大功能区。

#### 4.2.1 顶部区域

- **问候语**：根据当前时间动态显示（早上好/上午好/中午好/下午好/晚上好），附带 AI 助手卡通形象
- **问题清单卡片**：展示 AI 汇总的员工执行问题，点击「去查看」跳转管理端问题清单

#### 4.2.2 快捷问题入口

展示 3 条预设问题（「餐饮店在哪些环节可以用 AI 来干活？」等），点击后直接发送到对话流并触发 AI 回复。

#### 4.2.3 功能导航（底部功能区）

根据 `stage` 和 `isLoggedIn` 状态，功能按钮有三种显示状态：

| 状态 | 显示效果 | 点击行为 |
|------|----------|----------|
| 已登录 + Stage 2/3 | 正常显示，橙色图标 | 跳转对应功能页 |
| 未登录 | 显示但带锁图标 | 触发登录弹窗 |
| Stage 1 | 部分功能隐藏 | — |

功能按钮列表：工资日结、当前工作、AI 巡检、AI 菜单、门店 AI 模型、集团 AI 模型、碎片化培训。

#### 4.2.4 AI 对话流

**消息类型：**

- `user`：用户发送的文字消息，橙色气泡，右对齐
- `ai`：AI 回复，白色毛玻璃气泡，左对齐，底部带操作栏（语音播放/点赞/差评/刷新/分享 + 「内容由AI生成」提示）
- `follow-up`：AI 回复后自动追加的跟进问题气泡（3 条），点击后直接发送

**AI 回复机制（当前 Mock 实现）：**

1. 用户发送消息后，先显示「思考中」动画（500ms 延迟）
2. 匹配 `AI_REPLIES` 字典中的关键词，找到对应回复
3. 以流式打字效果逐字输出（每 30ms 一个字符）
4. 输出完成后，追加 `FOLLOW_UP_QUESTIONS` 中对应的 3 条跟进问题

**输入区状态：**

- 默认显示文字输入框（胶囊形，圆角 22px，毛玻璃白底）
- 左侧圆形按钮切换语音/键盘模式
- 有内容时右侧显示橙色发送按钮；无内容时显示麦克风占位图标
- 语音模式：「按住说话」按钮，按住后变橙色渐变 + 「松开发送」文字，松开后模拟语音识别结果

#### 4.2.5 Stage 3 触发条件

用户在 AI 对话中发送特定关键词（「试用」「申请」「体验」「合作」「价格」「费用」等）后，系统自动调用 `onEnterStage3()`，触发店铺信息采集弹窗。

### 4.5 AI 巡检

**页面：** `InspectionPage`

模拟 AI 辅助巡检流程，员工通过拍照上传，AI 自动识别问题并给出评分。

**状态流转：**

```
初始状态（showDetail=false）
  ──点击「开始巡检」──> 拍照上传状态
  ──AI 分析完成──> 显示巡检报告（showDetail=true）
```

**巡检报告内容：** 总分（0-100）、各检查项得分（环境卫生/食材新鲜度/服务规范/设备状态）、问题列表（含图片标注）、改进建议。

### 4.9 培训模块（员工端）

**页面：** `TrainingPage`

这是产品最核心的功能模块，采用 AI 对话流形式，将培训任务的接收和答题完全融入对话界面。

#### 4.9.1 页面进入方式

| 进入方式 | 初始状态 | 特殊行为 |
|----------|----------|----------|
| 从首页功能区点击「碎片化培训」 | 正常进入，AI 推送任务卡片 | 无 |
| 通过邀请链接进入（`fromInviteLink=true`） | 自动弹出「通用注册弹层」 | 注册完成后显示欢迎横幅 |

#### 4.9.2 对话流消息类型

| 消息类型 | 说明 | 视觉样式 |
|----------|------|----------|
| `ai` | AI 文字回复 | 白色毛玻璃气泡，左对齐，底部操作栏 |
| `user` | 用户回答 | 橙色气泡，右对齐 |
| `task-card` | 待完成培训任务卡片 | 白色卡片，含任务标题/题目数/截止时间/「开始培训」按钮 |
| `question` | AI 出题气泡 | 紫色渐变背景，含题目编号/进度/题目内容/关键点提示（可选） |
| `feedback` | AI 答题评价 | 绿色（正确）或橙色（需改进）背景，含得分/关键点分析 |
| `training-done` | 培训完成汇总 | 渐变背景，含总分/星级/「查看完整报告」按钮 |
| `completed-task` | 已完成的任务卡片 | 灰色背景，含完成时间/得分/「重新挑战」按钮（待实现） |

#### 4.9.3 答题状态机

```
isTrainingActive = false（未选定培训内容）
  ──点击任务卡片「开始培训」──> isTrainingActive = true
                              activeTask = 选中任务
                              currentQIdx = 0
                              attempts = 0

答题中（isTrainingActive = true）：
  用户发送答案
    ──评估答案──> 匹配关键点 ≥ 60% 或 attempts ≥ 2
      ──正确/通过──> feedback 气泡（正确）
                   ──还有下一题──> currentQIdx++, attempts=0
                   ──最后一题──> isFeedbackPhase = true（进入主观反馈）
      ──错误──> feedback 气泡（需改进）
               attempts++
               ──显示「查看提示」+「跳过此题」按钮（attempts ≥ 1）

  点击「查看提示」──> showHint = true，题目气泡展示关键点列表
  点击「跳过此题」──> 插入「跳过」气泡，进入下一题（跳过不计分）

isFeedbackPhase = true（主观反馈阶段）：
  AI 提问「请用自己的话说说这次培训的收获」
  用户提交反馈
    ──handleFinish()──> 计算总分，插入 training-done 气泡
                       isTrainingActive = false
                       activeTask = null
```

#### 4.9.4 评分规则

- 每题基础分 = 100 / 总题数
- 答对（首次）：满分
- 答对（使用提示后）：扣 20%
- 答对（2 次尝试后）：扣 30%
- 跳过：0 分
- 最终总分 = 各题得分之和（取整）

#### 4.9.5 底部输入区状态

| 状态 | 视觉表现 | 交互行为 |
|------|----------|----------|
| 未激活（`isTrainingActive=false`） | 整体半透明（opacity: 0.65），placeholder 显示「请先选择培训内容」 | 所有输入控件禁用 |
| 激活-文字模式 | 正常显示，胶囊输入框 | 可输入文字，Enter/发送按钮提交 |
| 激活-语音模式 | 显示「按住说话」按钮 | 按住录音，松开后模拟语音识别并自动提交 |
| 答错后 | 输入框上方出现「查看提示」+「跳过此题」两个胶囊按钮 | 点击对应操作 |

#### 4.9.6 注册弹层（通用注册弹层 / InviteRegisterModal）

当用户通过邀请链接进入时（`fromInviteLink=true`），自动弹出三步注册流程：

**Step 1：填写基本信息**
- 姓名（必填）
- 手机号（必填，11 位验证）
- 邀请人信息展示卡（邀请人姓名/职位/门店）

**Step 2：选择与邀请人的关系 + 选择职位**

这是核心设计点：接收者自行决定层级关系。

| 选项 | 视觉样式 | 后续职位列表 |
|------|----------|-------------|
| 我是 TA 的下级 | 橙色选中框 | 基层员工职位（服务员/厨师/收银员等） |
| 我是 TA 的上级 | 紫色选中框 | 管理层职位（区域经理/集团总监等） |

选择「下级」后，还需从下拉列表选择直属上级（支持多选，默认选中邀请人）。

**Step 3：注册成功**
- 显示注册成功动画
- 根据选择的关系显示不同文案：
  - 下级：「已加入 [门店名]，[邀请人] 是你的直属上级」
  - 上级：「已成为 [邀请人] 的直属上级，可在管理端查看其团队数据」
- 3 秒后自动关闭，显示欢迎横幅

**注册成功横幅（`showNewMemberBanner`）：**

## 5. 核心业务流程

### 5.1 培训任务全流程

管理者在培训管理页发起培训任务（选员工 → 选题目 → 设截止时间 → 生成通用邀请链接），员工在培训页 AI 对话流中接收任务卡片并逐题作答，答错可查看提示或跳过，全部完成后生成培训报告并同步至管理端。

![培训任务全流程时序图](https://d2xsxph8kpxj0f.cloudfront.net/310519663518710373/QXycpLBxSeowbm34BVNxpL/seq_training_flow_b989728b.png)

**关键状态说明：**

| 状态 | 触发条件 | 说明 |
|------|----------|------|
| `isTrainingActive = false` | 进入培训页默认状态 | 底部输入区禁用，显示「请先选择培训内容」 |
| `isTrainingActive = true` | 点击任务卡片「开始培训」 | 底部输入区激活，AI 开始出题 |
| `currentQuestionIndex` | 每次答对或跳过后递增 | 控制当前题目进度 |
| `attempts` | 每次答错后递增 | 大于 0 时显示「跳过此题」按钮 |
| `trainingPhase = 'feedback'` | 所有题目完成后 | 进入主观反馈阶段 |
| `trainingPhase = 'done'` | 提交主观反馈后 | 显示培训完成卡片 |

### 5.2 通用邀请注册流程

发起方生成不含身份标识的通用链接，接收方点击后在注册弹层中**自行选择与邀请人的层级关系**（我是TA的下级 / 我是TA的上级），系统根据选择自动将注册者归入正确的组织层级。发起方始终不知道接收者的身份，由接收者自主判断。

![通用邀请注册流程时序图](https://d2xsxph8kpxj0f.cloudfront.net/310519663518710373/QXycpLBxSeowbm34BVNxpL/seq_invite_register_13078e1a.png)

**关键设计决策：**

| 设计点 | 说明 |
|--------|------|
| 链接不含身份标识 | 参数仅含 `inviter` 和 `dept`，不含 `type=superior/subordinate` |
| 接收者自选关系 | Step2 提供「我是TA的下级」（橙色）和「我是TA的上级」（紫色）两个选项 |
| 职位列表动态切换 | 选下级：显示基层职位（服务员/厨师/收银员等）；选上级：显示管理层职位（区域经理/集团总监等） |
| 欢迎横幅颜色区分 | 下级注册：橙色横幅；上级注册：紫色横幅 |
| `hasSuperior` 自动更新 | 接收者选择「我是TA的上级」后，邀请人的 `hasSuperior` 自动变为 `true` |
