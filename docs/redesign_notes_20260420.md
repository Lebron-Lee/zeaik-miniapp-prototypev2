# 智爱客小程序文档重构笔记

## 一、文档重构目标

新版文档不再只服务于截图补齐，而是要让研发通过“回顾完整设计过程”理解当前小程序原型的**全部功能模块、页面边界、状态切换、重点交互与实施优先级**。文档表达应从“页面说明”升级为“场景驱动的设计说明”，尤其要把 AI 培训功能写成研发可直接照着拆任务的结构化说明。

## 二、当前产品的总体设计逻辑

| 维度 | 设计要点 | 研发理解方式 |
|---|---|---|
| 产品定位 | 面向餐饮连锁企业的 AI 运营管理小程序，聚焦培训智能体演示，同时串联工资日结、AI 巡检、AI 菜单、店面 AI、集团 AI 等能力 | 首页不是单纯导航，而是 AI 对话中枢与营销转化入口 |
| 交互骨架 | 采用 `MiniAppShell` 的单页状态切换模式，以 `appView` 控制页面视图 | 研发应按状态机实现，而不是按多 URL 页面理解 |
| 用户阶段 | Stage 1 游客体验、Stage 2 登录体验、Stage 3 深度转化 | 不同阶段决定首页入口是否需要登录拦截，以及是否进入店铺信息采集 |
| 重点闭环 | 发起培训 → 选对象 → 员工进入培训 → AI 问答 → 评分报告 → 管理追踪 | AI 培训是全产品的主线，其它模块承担认知、转化和辅助演示作用 |

## 三、需要在新版文档中覆盖的核心模块

| 模块 | 页面/文件 | 主要作用 | 需要强调的研发要点 |
|---|---|---|---|
| 首页 AI 对话中枢 | `HomePage.tsx` | 承接广告流量、产品咨询、快捷入口、首页内发起培训 | 快捷问题、流式回复、底部输入区、首页培训卡片、录音与资料沉淀 |
| 当前工作 | `CurrentTaskPage.tsx` | 员工工作清单与培训任务承接页 | 任务标签、培训任务迁移到工作场景、问题清单/任务大厅三类内容 |
| 抽屉页 / 培训中心 | `DrawerPage.tsx` | 聚合“我发起的 / 我参加的”培训任务与会话记录 | 作为二级入口，不替代主培训链路 |
| 工资日结 | `DailySalaryPage.tsx` | 展示员工收入与绩效反馈 | 从首页进入，反向联动当前工作 |
| AI 巡检 | `InspectionPage.tsx` | 演示拍照识别、巡检评分、整改建议 | 适合作为 AI 能力展示，不是本轮主线 |
| AI 菜单 / 店面 AI / 集团 AI | 多个介绍页 | 作为产品矩阵认知页面，支撑销售与产品理解 | 重点写作用途、入口和与培训模块的关系 |
| 培训主页（员工端） | `TrainingPage.tsx` | 接收任务、无感注册、开始培训、查看历史完成情况 | 写清从邀请链接进入、注册、任务列表、开始培训的承接关系 |
| 培训答题页 | `TrainingAnswerPage.tsx` | AI 发题、答题、提示、跳过、语音/文字回答 | 重点写题目状态机和错误分支 |
| 培训报告页 | `TrainingReportPage.tsx` | 汇总得分、评级、AI 评语、薄弱专项练习 | 重点写评分结构与练习入口 |
| 培训管理页 | `TrainingManagerPage.tsx` | 发起培训、完成率看板、问题清单 | 重点写 4 步创建流程和管理视图 |
| 组织架构选择页 | `OrgTreePage.tsx` | 按层级选对象、邀请成员、组织增长承接 | 重点写“集团 / 管理层 / 店长”两层选择模型 |
| 组织注册页 | `OrgRegisterPage.tsx` | 邀请成员后的注册承接 | 作为组织增长的补充页说明 |

## 四、AI 培训模块需要重写为研发实施说明的内容骨架

| 章节 | 必须说明的内容 |
|---|---|
| 培训功能定位 | 为什么产品要把培训做成 AI 对话，而不是传统答题页 |
| 用户角色 | 发起者、培训参与员工、被邀请注册成员、上级管理者各自看到什么 |
| 页面链路 | 首页发起 / 抽屉进入 / 邀请链接进入 / 管理端进入的不同起点 |
| 关键页面 | 培训主页、答题页、提示分支页、报告页、管理结果页、组织选择页 |
| 跳转关系 | 哪个按钮跳哪一页，页面返回后保留什么状态 |
| 状态机 | 任务待开始、题目进行中、答错提示、跳过、反馈题、完成报告 |
| 评分机制 | 首次答对、使用提示后答对、多次尝试后答对、跳过的不同计分规则 |
| 场景支撑 | 新员工培训、门店标准化培训、会议现场演示、培训补考、组织增长邀请 |
| 研发拆分建议 | 哪些是前端状态问题，哪些需要后端数据结构支撑 |

## 六、文档表达方式建议

新版文档应按“**设计目标 → 页面模块 → 使用场景 → 跳转关系 → 研发实现提示**”的顺序组织。对于 AI 培训章节，应在每个页面说明后增加一张表，明确页面入口、主要状态、核心字段、下一个跳转节点以及研发实现重点，使研发不必反复对照代码文件才能理解产品设计意图。
