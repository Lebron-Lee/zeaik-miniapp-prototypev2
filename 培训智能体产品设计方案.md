# 智爱客「培训智能体」产品设计方案

> **版本**：v1.0 · **日期**：2026-04-12 · **状态**：待确认

---

## 一、产品定位与设计目标

### 1.1 核心定位

「培训智能体」是智爱客面向餐饮行业的 AI 驱动培训应用，以**对话式语音答题**为核心交互范式，替代传统集中培训、老带新、会议解读等低效模式。其本质是一个**随时随地、按需触达、闭环验证**的移动端培训工具，嵌入员工日常工作流中。

产品对标「豆包」等对话式大模型应用的交互体验，以情绪友好、智能引导为设计基调，让员工在轻松愉悦的氛围中完成知识内化，而非被动接受考核压力。

### 1.2 核心设计目标

| 目标维度 | 具体描述 |
|---|---|
| **触达效率** | 通过小程序分享链接，任意一级发起培训即可拉动全员无感注册，低成本构建组织架构 |
| **培训效果** | 答对才进下一题，AI 持续追问关键知识点，确保真正掌握而非蒙混过关 |
| **情绪价值** | 全程 AI 陪伴式引导，鼓励性语言、进度激励、结束表扬，降低员工抵触感 |
| **管理可见** | 店长/集团实时查看完成率、问题清单排名、薄弱知识点，支持精准专项指导 |
| **组织沉淀** | 每次培训数据自动归属上级，逐步沉淀全公司/全集团员工能力画像 |

## 二、用户角色与组织架构

### 2.1 金字塔式多级组织

系统采用**纯金字塔架构**，每个用户注册时必须绑定上级，上级可以是多个（允许矩阵管理场景）。所有人均可成为上级角色，顶级角色（集团总部）创建培训主题，各级可选择继承上级任务分发，或自行创建任务下发。

```
集团总部（顶级）
  └── 区域经理
        └── 店长
              └── 班组长
                    └── 员工（末级）
```

**关键设计原则**：
- 任意一级均可发起培训，上级自动对齐（发起人的上级同步可见结果）
- 支持跨层级穿透下发（集团可直接推送至全体员工）
- 上级可查看所有下级的培训完成情况和得分

### 2.2 用户角色权限对照

| 角色 | 创建题库 | 发起培训 | 查看下级结果 | 接受培训 | 查看自身结果 |
|---|:---:|:---:|:---:|:---:|:---:|
| 集团管理员 | ✅ | ✅ | ✅（全员） | — | — |
| 区域经理 | ✅ | ✅ | ✅（本区域） | ✅ | ✅ |
| 店长 | ✅ | ✅ | ✅（本店） | ✅ | ✅ |
| 班组长 | — | ✅（转发） | ✅（本组） | ✅ | ✅ |
| 员工 | — | — | — | ✅ | ✅ |

### 2.3 无感注册与组织拉新机制

培训任务通过小程序分享链接传播。未注册用户点击链接后，系统引导其完成**最简注册**（手机号+选择上级），注册即自动加入组织架构，完成培训任务。这一机制使企业无需提前录入人员信息，通过第一次培训发起即可完成全公司/全集团的组织架构梳理与用户数据注册。

## 三、题库体系设计

### 3.1 五大题库类型

| 题库类型 | 适用场景 | 发起时机 | 内容特征 |
|---|---|---|---|
| **新员/临时工题库** | 替代集中培训、老带新 | 入职当天/次日 | 基础规范、门店常识、安全须知 |
| **制度与文化题库** | 替代文件发布会议解读 | 制度更新、文件发布后 | 制度条款理解、文化认同 |
| **技能题库** | 替代岗位专线培训 | SOP变更、新菜品研发后 | 操作步骤、服务礼仪、菜品知识 |
| **专项题库** | 替代日常例会专项强调 | 日常经营问题发现后 | 针对性问题纠偏、案例分析 |
| **自命题题库** | 个性化临时培训需求 | 随时 | 店长自由创建，灵活定制 |

### 3.2 题目形式

当前阶段优先支持以下题型，后续可扩展：

- **开放式问答**（主要形式）：员工用语音/文字自由作答，AI 判断关键知识点覆盖情况
- **选择题**：单选/多选，适合制度类、规范类题目
- **情景判断题**：描述一个服务场景，员工判断处理方式是否正确

### 3.3 题库建设路径

**近期（MVP阶段）**：优先使用现有数据库中的固有题库跑通答题流程，不开发实时解析功能。

**中期**：开发文件上传解析功能，支持将制度文件、SOP文档等自动解析为结构化题目，降低题库建设成本。

## 四、核心答题流程设计

### 4.1 完整答题流程

```
店长发起培训任务
      │
      ▼
选择题库类型 + 选定1-5道题目
      │
      ▼
推送至员工手机端（小程序消息通知）
      │
      ▼
员工接收任务 → 进入答题界面
      │
      ▼
┌─────────────────────────────────────────┐
│           单题答题循环                    │
│                                         │
│  AI 朗读/展示题目                        │
│       │                                 │
│       ▼                                 │
│  员工语音/文字作答（多轮交流）             │
│       │                                 │
│  ┌────┴────┐                            │
│  │ 答对？  │                            │
│  └────┬────┘                            │
│    是 │ 否                              │
│       │  └─→ AI 引导追问（关键知识点）   │
│       │         │                       │
│       │    员工可申请查看提示答案         │
│       │    （开始输入时自动关闭答案）     │
│       │         │                       │
│       │    继续作答直至答对              │
│       ▼                                 │
│  进入下一题（含情绪激励语）              │
└─────────────────────────────────────────┘
      │
      ▼
固定题目：「对本次培训内容的看法和意见」
      │
      ▼
AI 综合分析 → 输出培训报告（5分制评分 + 简要分析）
      │
      ▼
员工查看结果 → 可发起薄弱知识点专项练习
```

### 4.2 AI 交互行为规范

**答对时**（情绪激励示例）：
> "太棒了！这个知识点你掌握得很扎实，继续保持这个状态！"

**答错时**（引导式追问，而非机械报错）：
> "你说到了两个关键点，还有一个重要的细节没有提到，再想想在这个场景下还需要注意什么？"

**多次答错时**（提示答案前）：
> "没关系，这个知识点确实有点难记。我给你看一下参考答案，仔细看完后关闭，再用自己的话说一遍给我听。"

**答案提示机制**：
- 员工主动点击「查看提示」才展示答案
- 员工开始语音输入或文字输入时，答案自动收起
- 答案展示期间无法提交答案（强制记忆）

**培训结束总结**（情绪价值输出示例）：
> "今天的培训你完成得很认真！5道题全部答对，得了4.2分。其中「茶水区清洁标准」这个知识点你回答得特别好。「礼仪接待流程」还有一个细节需要再加强，建议明天专项练习一下。加油，你越来越专业了！"

### 4.3 固定反馈题目

每次培训结束前，强制增加一道开放式题目：

> **「请用自己的话说说，这次培训让你印象最深的是什么？有什么想法或者建议？」**

此题不计入评分，但 AI 会对员工反馈进行情感分析，供店长了解员工培训体验和情绪状态。

## 五、培训报告设计

### 5.1 员工端报告（简洁版）

培训结束后即时生成，内容包括：

- **综合评分**：5分制，大字展示，配情绪化评语（如「优秀学员」「继续加油」）
- **题目掌握情况**：每道题一行，标注「掌握✅」或「需加强⚠️」
- **AI 个性化建议**：1-2句话，指出最需要加强的知识点
- **专项练习入口**：针对未掌握题目，一键发起专项练习

### 5.2 店长端看板（管理视角）

| 模块 | 内容 |
|---|---|
| **完成率总览** | 培训任务完成率 + 任务完成率（双维度） |
| **问题清单排名** | Top 10 高频错误知识点，支持下钻查看明细（哪些员工在哪道题上出错） |
| **员工能力画像** | 每位员工的历史培训得分趋势、薄弱知识点标签 |
| **情绪监控** | 基于员工反馈题的情感分析，识别低情绪员工 |
| **服务质量关联** | 培训得分与营业额、服务评分的相关性分析（中期功能） |

## 六、页面模块规划

### 6.1 员工端页面清单

| 页面 | 功能描述 |
|---|---|
| **培训任务列表** | 待完成/已完成任务卡片，显示题目数量、截止时间、发起人 |

| **答案提示弹层** | 半屏弹出，展示参考答案要点，输入时自动收起 |

| **专项练习页** | 针对薄弱知识点的定向练习，与答题主界面复用 |

| 页面 | 功能描述 |
|---|---|
| **发起培训** | 选择题库类型 → 选题（1-5题）→ 选择下发对象 → 设置截止时间 → 发送 |

| **员工能力画像** | 单个员工的培训历史、得分趋势、薄弱标签 |
| **题库管理** | 查看/创建/编辑题目，支持文件上传解析（中期） |

## 七、技术实现关键点

### 7.1 AI 能力要求

- **语音识别（ASR）**：实时将员工语音转为文字，支持普通话及方言（可选）
- **答案评判引擎**：基于关键知识点匹配，而非精确字符串比对，允许员工用自己的语言表达
- **多轮对话管理**：维护单题的对话上下文，AI 根据员工历次回答动态调整追问策略
- **情感分析**：对员工反馈题的文字进行情感极性分析

### 7.2 消息推送

微信小程序订阅消息（需用户授权）用于培训任务推送通知，需在发起培训时引导员工完成消息订阅授权。（待确认微信生态消息推送条数限制 @薛君浩）

### 7.3 数据归属与隐私

员工答题数据归属其直属上级可见范围，跨级查看需有对应权限。员工个人得分数据不对同级员工公开，仅管理层可见排名。

## 八、MVP 范围建议

考虑到快速验证核心价值，建议 MVP 阶段聚焦以下功能：

**✅ MVP 必须包含**
- 员工端：收到任务通知 → 语音/文字答题 → 查看培训报告
- 店长端：选题 → 发起培训 → 查看完成率 + 问题清单
- AI 能力：关键知识点判断 + 情绪激励语言 + 简单报告生成
- 组织架构：注册时绑定上级，支持2-3级层级

**⏳ 中期迭代**
- 文件上传自动解析题目
- 员工能力画像与薄弱知识点专项练习
- 情绪监控看板
- 跨层级穿透管理

**🔮 长期规划**
- 培训效果与营业额/服务质量关联分析
- AI 眼镜端任务发送（硬件集成）
- 自适应题目难度（根据历史表现动态调整）

## 九、待确认事项

在进入原型开发前，以下问题需要与产品/技术团队确认：

1. **消息推送限制**：微信小程序订阅消息每次可推送几条？是否满足批量任务推送需求？（@薛君浩 待办）
2. **AI 评判精度**：开放式语音答题的关键知识点匹配，是否接入外部大模型 API（如智爱客自有 AI 中台）？
3. **语音输入方案**：是否使用微信小程序原生语音识别，还是接入第三方 ASR 服务？
4. **题库初始数据**：MVP 阶段使用哪些现有题库数据？需要产品/运营提供初始题目内容
5. **组织架构层级上限**：金字塔最多支持几级？是否需要支持矩阵式（一个员工有多个上级）？
6. **评分规则细化**：5分制的评分算法如何设计？答对次数、答题轮数、用时是否都纳入评分？
